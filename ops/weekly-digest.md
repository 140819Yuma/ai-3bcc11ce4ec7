# 週次AIリサーチダイジェスト 運用キット

目的: 「AIエージェント動向のリサーチ→index.htmlへの蓄積」を、
**週1回・コピペ1行**で回せるようにする。将来的には完全自動化する。

---

## いますぐ使える運用(手動トリガー・コスト追加なし)

毎週月曜、`AI作業` フォルダで Claude Code を開いて以下を貼るだけ:

```
週次ダイジェストを実行して
```

これで `ai-research` スキルの週次ダイジェストモードが発動し、
調査→絞り込み→index.htmlの#logへの追記→QAまで一括で実行される。

**人間がやること(5分):**
1. 追記されたダイジェストをブラウザで開いて読む
2. プロトタイプに影響がある項目だけKokiさんに共有
3. 気になるトピックがあれば「◯◯を深掘りして」と追加指示

---

## 忘れない仕組み(リマインダー)

macOS標準のリマインダーに週次登録する(無料・確実):

```bash
osascript -e 'tell application "Reminders" to make new reminder with properties {name:"週次AIダイジェスト実行(AI作業フォルダでClaude起動)", due date:date "Monday 10:00"}'
```

またはカレンダーに「月曜10:00 週次ダイジェスト」を繰り返し登録。

---

## 将来の完全自動化(要検討・実行前に必ず判断)

Claude Code の `/schedule` 機能でクラウドエージェントとして毎週自動実行できる。
ただし **クラウド実行はAPI利用分の課金が発生しうる** ため、
導入するときは以下を確認してから:

1. 現在のプラン(Pro/Max)でscheduled agentsが使えるか、追加課金の有無
2. 週1回・1回あたりの想定コスト
3. 自動実行結果を人間がレビューするタイミング

導入する場合のコピペ用指示(Claude Codeに貼る):

```
/schedule 毎週月曜10時に、AI作業フォルダで「週次ダイジェストを実行して」を実行する定期タスクを作って。作成前に課金の有無を教えて。
```

---

## 品質が落ちてきたときのチューニング

- トピックが多すぎる → SKILL.mdの「最大5トピック」を3に減らす
- 関係ない話題が混ざる → SKILL.mdの調査対象リストを具体化する
- index.htmlが重くなった → `research/YYYY-MM.html` への月別分割を指示する
