# AIリサーチ スキル

## スキルの概要

このスキルは、AIエージェント開発に関するリサーチを実施し、その結果を `index.html` に蓄積していくためのワークフローを定義する。
Yumaとのプロジェクトで「調べたこと・気づいたこと・設計決定」を体系的にまとめるための共通手順。

---

## 発動タイミング

以下のような依頼があった場合にこのスキルを使用する：

- 「〇〇について調べて、HTMLに追加して」
- 「リサーチしてまとめて」
- 「このPDF/動画の内容をHTMLにまとめて」
- 「〇〇のセクションをindex.htmlに追加して」
- 「AIリサーチを実行して」

---

## 実行手順

### STEP 1: インプットを把握する

まず何を調査するかを明確にする。ソースは以下のいずれか：

1. **ユーザーのメッセージ**（「〇〇について調べて」）
2. **PDFファイル**（`Read` ツールでフォルダ内のPDFを読む）
3. **URL・YouTube動画**（`WebFetch` でタイトル・説明取得、`WebSearch` で補完）
4. **既存の `index.html`**（現在の内容を把握してから更新）

### STEP 2: 現在の index.html を確認する

```bash
# 現在のHTMLの構造を把握
Read /Users/kaminagayuma/Library/CloudStorage/OneDrive-u-aizu.ac.jp/デスクトップ/AI作業/index.html
```

- 既存のセクション一覧を把握
- 新しいコンテンツをどこに追加するか決定
- 重複がないか確認

### STEP 3: リサーチを実施する

調査方法は並列で行う：

```
WebSearch（日本語・英語） + WebFetch（詳細ページ）を同時実行
```

調査対象の例：
- アーキテクチャパターン・設計手法
- フレームワーク比較（LangGraph / CrewAI / AutoGen 等）
- API仕様・料金・制限値
- 実装コードサンプル
- ベストプラクティス・アンチパターン

### STEP 4: index.html に追記・更新する

#### 追記ルール

1. **既存セクションへの追記**：関連するセクションに情報を加える
2. **新しいセクション追加**：テーマが独立している場合は `<section id="新id">` を追加
3. **進捗ログの更新**：`#log` セクションに日付付きのエントリを追加

#### HTMLの統一スタイルルール

- ダークテーマ（`--bg: #0d1117` ベース）を維持
- セクション構造：`.sec-tag` → `.sec-title` → `.sec-sub` → コンテンツ
- カードグリッド：`.card-grid` > `.card`
- コードブロック：`.code-wrap` > `.code-head` + `<pre>`
- テーブル：`.tbl-wrap` > `<table>`
- 注意書き：`.callout.c-info/.c-warn/.c-tip/.c-danger`
- ナビゲーションの `<ul>` にアンカーリンクを追加

#### 追記の際に含めるべき情報

| 種類 | 含める内容 |
|------|-----------|
| 技術調査 | 仕組み・メリット・デメリット・実装コード例 |
| 比較 | テーブル形式で並列比較 |
| 進捗ログ | 日付・完了事項・気づき・次のアクション |
| API情報 | 無料枠の制限値・推奨モデル・コスト感 |
| 設計決定 | なぜその選択をしたか・トレードオフ |

### STEP 5: 確認

- HTMLファイルのサイズが適切か確認
- リンクが正しいか確認
- セクション間のナビゲーションリンクが追加されているか確認

## 注意事項

- PDFやURLを追加リサーチする際は、WebSearch と WebFetch を並列実行して効率化する
- 既存のHTMLスタイル（CSS変数・クラス名）を壊さないように追記する
- 情報の鮮度（取得日）をフッターまたはコメントに記載する
- 機密情報（APIキーなど）はHTMLに直接書かない
- 長くなりすぎる場合は別ファイル（`research/topic-name.html`）に分割することも検討する
