## スラッシュコマンド一覧

### プロジェクト初期化

| コマンド | 説明 | 使用例 |
|---------|------|--------|
| `/init` | `CLAUDE.md` を生成してプロジェクトを初期化 | `/init` |
| `/memory` | CLAUDE.md メモリファイルを編集・管理 | `/memory` |
| `/mcp [reconnect]` | MCP サーバー接続を管理・認証 | `/mcp reconnect openai` |
| `/agents` | サブエージェントの設定を管理 | `/agents` |

---

### セッション管理

| コマンド | 説明 | オプション |
|---------|------|-----------|
| `/clear [name]` | 新しい会話を開始（プロジェクトメモリは保持） | 前の会話に名前を付与できる |
| `/resume [session]` | 過去の会話を再開 | セッション ID または名前を指定 |
| `/branch [name]` | 現在の会話を複製（元の会話も保持） | 別の方向性を試すときに使用 |
| `/fork <指令>` | バックグラウンドサブエージェントを起動 | 指令を渡して並列実行 |
| `/cd <path>` | セッションを新しいディレクトリに移動 | 設定キャッシュは保持される |
| `/add-dir <path>` | 現在のセッションにアクセスディレクトリを追加 | 設定は自動発見されない |

---

### モデル・リソース管理

| コマンド | 説明 | オプション |
|---------|------|-----------|
| `/model [model]` | AI モデルを切り替え | `opus` / `sonnet` / `fable` など |
| `/effort [level\|auto]` | 推論の深さを調整 | `low` / `medium` / `high` / `xhigh` / `max` / `ultracode` |
| `/advisor [model\|off]` | 第二のモデルを参考として使用 | `opus` / `sonnet` / `fable` |
| `/fast [on\|off]` | ファストモードを切り替え（コスト削減） | — |
| `/config [key=value]` | 設定を開く・変更 | `theme=dark`, `model=sonnet` など |
| `/usage` | セッションコスト・使用量を表示 | `/stats` `/cost` は別名 |

### コンテキスト管理

| コマンド | 説明 | 用途 |
|---------|------|------|
| `/context [all]` | コンテキスト使用量を可視化 | 最適化提案を表示 |
| `/compact [instructions]` | 会話を要約してコンテキストを解放 | フォーカス指示を渡すことも可能 |

### コード変更・レビュー

| コマンド | 説明 | オプション |
|---------|------|-----------|
| `/plan [description]` | プランモードに入る | 大きな変更の前に確認 |
| `/diff` | 未コミット変更をインタラクティブ表示 | ← → で turn 間を切り替え |
| `/code-review [level] [--fix] [--comment]` | diff をレビュー | `low` / `medium` / `high` / `max` / `ultra` |
| `/simplify [target]` | クリーンアップ機会を検出・適用 | コード簡潔化 |
| `/review [PR]` | GitHub PR をレビュー | PR 番号または URL で指定 |
| `/security-review` | セキュリティ脆弱性を分析 | git diff をスキャン |

## 典型的なワークフロー

```
【初回セッション】
/init        → CLAUDE.md を生成してプロジェクトを登録
/memory      → 設定を編集
/mcp         → MCP サーバーを接続
/agents      → サブエージェントを設定

【タスク実行中】
/plan        → 大きな変更の前にプランモードで確認
/model       → モデルを切り替え（重い処理は opus など）
/effort      → 推論の深さを調整
/context     → コンテキスト使用量を確認
/compact     → 必要に応じて会話を要約・解放

【変更レビュー前】
/diff        → 変更内容を確認
/code-review → バグ・品質を検査
/simplify    → コードのクリーンアップ
/security-review → セキュリティを確認

【リリース前】
/review [PR] → GitHub PR をレビュー
/verify      → アプリで実際に動作確認
git commit & push
```

## カスタマイズ方法

### キーバインディング設定ファイル

```bash
/keybindings
```

を実行すると `~/.claude/keybindings.json` が開く。

**設定例：Ctrl+E を外部エディタに割り当て**

```json
{
  "$schema": "https://www.schemastore.org/claude-code-keybindings.json",
  "bindings": [
    {
      "context": "Chat",
      "bindings": {
        "ctrl+e": "chat:externalEditor",
        "ctrl+u": null
      }
    }
  ]
}
```

## コマンド発見のコツ

1. **`/` だけ入力する** — 利用可能なすべてのコマンドを表示
2. **`/ + 文字` で絞り込む** — 例：`/mo` → `/model`, `/mobile` が候補に表示
3. **`/help`** — ヘルプメニューを表示
4. **`/keybindings`** — キーバインディング設定ファイルを開く
